CRA user-centred design guide

Subway

The subway navigation pattern is used to help people accomplish a single task that is broken into steps. It is ideal for presenting services and processes.

On this page

  • When to use
  • What to avoid
  • Content and design
  • Examples
  • Research and blog posts

When to use

  • To break up long and complex content into pages that each focus on a specific step or answer people need before moving on to the next step or section
  • To provide navigation between the related pages that reflects the journey that the user would go through

What to avoid

  • Do not use for any reason except to guide the user through a single task that is broken into steps
  • Do not give the steps numbers
    • Testing has shown that Canadians are less likely to get to the section they need if the elements are numbered
  • Do not use side-steps unless a step:
    • Has additional steps that the user must take
    • Has content for specific audiences or scenarios that doesn’t apply to all users
  • Do not use “previous” buttons for user to navigate backwards in the process, as you want to encourage users to move forwards

Content and design

Before you start, plan the architecture for your content. Keep the number of main task steps to a reasonable number (aim to have no more than 8) and determine if you need any side-step pages.

There are three page types within the subway pattern:

  • First level of information architecture (IA):
    Index page (mandatory)

  • Second level of information architecture (IA):
    Step pages (mandatory)

  • Third level of information architecture (IA):
    Side-step pages (optional)

    There are 2 options:

    • Non-connected page

    • Connected page

Steps

  1. Determine the information architecture

    The flow of content creates a "happy path" for the user to navigate the content in the following order:

    1. Start by introducing the concept

      Create a small introductory statement that explains what the product or activity is.

      • Typical design

        This is the most common. Typically the introductory statement fits on the index page. It is a short paragraph or two, and is the first thing to appear on the index page, above the subway navigation.

        For example:

        • Canada Dental Benefit

        For example:

        • File information returns (slips and summaries)
      • Complex design

        Sometimes the concept is too complex to reasonably describe on the index page. When this happens, feature basic details on the index page and use the first step page in the subway to provide more detail.

        For example:

        • Disability tax credit (DTC)
          • What is the DTC

        For example:

        • Scientific Research and Experimental Development (SR&ED) tax incentives
          • What are SR&ED tax incentives

      (Details here)

    3. Then explain what the user has to do

      (Details here)

    4. Wrap-up with any activities that occur after the core activity is done

      (Details here)

    5. Provide contact details (optional)

      (Details here)

  2. Create the index page

    When to use

    Use as the landing page for a set of related pages that use the subway navigation.

    How to implement
    • Provides a unique H1 followed by a short description of the service or process
    • Includes a list of links to each step that follows the Services and information - Canada.ca design pattern requirements for link text and link descriptions
    • The steps are linked together in the centre of the page using a subway pattern
    • This page is the anchor link in the breadcrumb for all step pages

    Supportive information is an optional element that can be included if necessary.

    Supportive information (optional)

    Supportive information is an optional element. It includes links to information that is associated with the content but not part of a task step. It should be avoided whenever possible, since it can distract a user from the task and pull them away from the task that they should be following. Do not use if the supportive information only applies to a few pages only and not the entire task itself.

    How to implement

    • The links can appear on the main page (after the subway menu), and below the subway menu in desktop, or at the bottom of the page in mobile
    • They should be clearly separated from the subway menu and de-emphasized, for example by using a smaller font size
    • There should be a clear heading and link text that doesn’t distract users from the task
    • Use to link to information that supports the task (for example statistics or a database), but not to other related information that the user doesn’t need (for example a legislative act)

  3. Create all the step pages

    When to use

    Use on each page tied by the subway navigation.

    How to implement

    • This page’s title element must include both this page name and the index page name
      • Example: <title>Step page – Index page – Canada.ca</title>
    • Provide a subway navigation menu on the right side of the page that:
      • Has a header called “Sections”
      • Includes a subway pattern that connects the step pages together

        • Fill in the circle for the page that the user is on

      • Has plain language, task-based content that helps the user to complete the step

        • Can be combined with other relevant patterns, such as the interactive questions pattern to help guide the user through the step

      • Includes a button on the bottom right with the name of the next step and a right pointing arrow

  4. Create any side-step pages, if needed

    When to use

    Side-step pages are an optional element that can be added when needed to break up a large step into smaller steps or to outline a side journey for specific audiences related to the main step.

    How to implement

    • The breadcrumb of these pages should link to both the main index page and the parent step page that contains the side-steps (e.g. Index page > Step page > side-step page).
    • This page’s title element must include both this page name and the index page name, and not the step page name (e.g. <title>side-step page – Index page</title>). This should be taken into consideration when naming the side-step pages.
    • The task step page which introduces the side-steps should be brief in information and proceed straight to the navigation to the side-steps. Since the page already has links to the side-step pages in the subway menu on the right, there is no need to show the side-step navigation in the main navigation on this page.
      • Alternatively, the side-step pages can be linked in context of the parent step page's content

    There are two types of side-steps:

    • Non-connected side-steps
    • Connected side-steps
    Non-connected side-steps

    How to implement

    • Navigation with pagination should lead back (Previous) to the parent step page, and should lead to (Next) the next step item as the side-step pages are not in sequential order

    Research and blog posts

    This pattern was developed with CRA during an optimization project on the Canada Child Benefit. It has been further tested on numerous other benefit programs, including the covid-19 benefits. It works well on mobile devices and is an effective way to show the relationship between related pages.
