Subway – CRA user-centred design guide

CRA user-centred design guide

Subway

The subway navigation pattern is used to help people accomplish a single task that is series of related tasks that are broken into steps. It is ideal for presenting specific services and processes.

On this page

  • When to use
  • What to avoid
  • Content and design
  • Research and blog posts

When to use

  • To break up long and complex content into pages that each focus on a specific step or answer people need before moving on to the next step or section
  • To provide persistent navigation between the related pages that reflects the journey that the user would typically go through

What to avoid

  • Do not use for any reason except to guide the user through a single task that is series of related tasks that are broken into steps
  • Do not drop the subway menu from one pager to another, as it is persisent and must appear on every page
  • Do not give the steps numbers
    • Testing has shown that Canadians are less likely to get to the section they need if the elements are numbered
  • Do not use side-steps unless a step:
    • Has additional steps that the user must take
    • Has content for specific audiences or scenarios that doesn’t apply to all users

Content and design

Before you start, plan the architecture for your content. Keep the number of main task steps to a reasonable number (aim to have no more than 8) and determine if you need any side-step pages.

There are three page types within the subway pattern:

Steps

  1. Determine the information architecture

    The flow of content across the various subway pages creates a "happy path" for the user to view the content in a logical progression.

    1. Introduce the concept

      Create a small introductory statement that explains what the product or activity is.

      • If its easy and brief to describe

        Typically the introductory statement fits on the index page. It is a short paragraph or two, and is the first thing to appear on the index page, above the subway navigation.

        For example:

        • Canada Dental Benefit
        • Canada Workers Benefit
      • If it's complicated to describe

        Sometimes the concept is too complex to reasonably describe on the index page. When this happens, feature basic details on the index page and use the first step page in the subway to provide more detail.

        For example:

        • Index page has a small teaser: Disability tax credit (DTC)
          • First subway circle has the full description: What is the DTC
        • Index page has a small teaser: Scientific Research and Experimental Development (SR&ED) tax incentives
          • First subway circle has the full description: What are SR&ED tax incentives
    2. Explain who it's for

      This is often where eligibility information goes. Common label items include:

      • Who can apply
      • Who can claim
      • How much you can get
      • Determine if you need to register
      • Periods you can apply for
      • Expenses you can claim
      • Which of your employees qualify
      • Who is eligible
        Use "Who is eligible" as a last resort

        Use the plain language terms "apply" or "claim" instead of "eligble". However, in rare situations, "eligible" may be used if the other terms might cause user confusion for that product or service.

    3. Explain what core activity the user has to do

      This where the core activity goes. The core activitiy is typically the main apply, claim, register, file, or submit task. Common label items include:

      • Get ready to apply
      • When to apply
      • Calculate your subsidy amount
      • Calculate your expenses
      • How to apply
      • How to claim
    4. Wrap-up with any activities that occur after the core activity is done

      This where the activities go that occur after the core activity has been done. Common label items include:

      • Validating your application
      • Our review and decision
      • After you apply
      • After you claim
      • When and how to get your payment
      • Impact on your taxes
      • Change or cancel your claim
      • Return a payment
      • Keep getting your payments
    5. Provide contact details (optional)

      If there are specific phone numbers or mailing addresses associated, it is optional to create a contact page. The label is usually:

      • Contact us

  2. Create the index page

    Screenshot: Index page

    Use as the landing page for a set of related pages that use the subway navigation.

    title tag
    [Service name] - Canada.ca
    h1 heading
    Provides a unique H1 within the context of the entire Canada.ca website. It must be specific enough that it does not overlap any other Governement of Canada page.
    Breadcrumb
    This page is the parent page, or level-up page, for all subsequent step pages
    Content area
    • Must include a short description of the service or process below the main heading
    • Must include a subway navigation that includes both the link and link description

    • May include supportive links to products or pages outside the subway. These links:

      • Should be avoided whenever possible, since it can distract a user from the task and pull them away from the task that they should be following
      • Appear below the subway menu, under its own heading called "Related"
        • If the supportive links can be bundled together into groups, you can rename "Relatede" and placed them under specifically named headings
      • Apply to the entire service or process
    Subway navigation
    Includes a list of links to each step that follows the doormats pattern for link text and link descriptions
    Pagination buttons
    • Backwards: None
    • Forwards: None

  3. Create all the step pages

    Screenshot: Step page

    Use on each page tied by the subway navigation.

    title tag

    This page’s title element must include both this page name and the index page name

    • Example: <title>Step page – Index page – Canada.ca</title>
    h1 heading
    Since the service name will already be featured with the two H1 design, there is not need to duplicate text or terms around in the service name. The step page should be as concise as possible. For example:
    • Do: How to apply
    • Don't: How to apply for [x]
    Breadcrumb
    The subway's index page must be the parent page in the breadcrumb
    Content area
    Place content into the page. Common design components include:
    • Table of contents
    • Headings
    • Lists
    • Links
    Subway navigation
    Supportive links
    Pagination buttons
      • Backwards: None

        Do not use “previous” buttons for user to navigate backwards in the process, as you want to encourage users to move forwards

      • Forwards: ???
      None

      • Has a header called “Sections”

  4. Create any side-step pages, if needed

    When to use

    Side-step pages are an optional element that can be added when needed to break up a large step into smaller steps or to outline a side journey for specific audiences related to the main step.

    How to implement

    • The breadcrumb of these pages should link to both the main index page and the parent step page that contains the side-steps (e.g. Index page > Step page > side-step page).
    • This page’s title element must include both this page name and the index page name, and not the step page name (e.g. <title>side-step page – Index page</title>). This should be taken into consideration when naming the side-step pages.
    • The task step page which introduces the side-steps should be brief in information and proceed straight to the navigation to the side-steps. Since the page already has links to the side-step pages in the subway menu on the right, there is no need to show the side-step navigation in the main navigation on this page.
      • Alternatively, the side-step pages can be linked in context of the parent step page's content

    There are two types of side-steps:

    • Non-connected side-steps
    • Connected side-steps
    Non-connected side-steps

    How to implement

    • Navigation with pagination should lead back (Previous) to the parent step page, and should lead to (Next) the next step item as the side-step pages are not in sequential order

    Non-connected side-steps

    When to use

    Break up a large step into smaller pieces of mutually exclusive information, such as unrelated sub-tasks or content that relates to a specific audience.

    Step page for non-connected side-steps

    How to implement

    In the main content area, the task step index page uses the Services and information - Canada.ca design pattern, 2 across with links and doormats

    • Expand the right subway menu to include the side-steps
    • Do not include a connecting line between the side-steps

Research and blog posts

This pattern was developed with CRA during an optimization project on the Canada Child Benefit. It has been further tested on numerous other benefit programs, including the covid-19 benefits. It works well on mobile devices and is an effective way to show the relationship between related pages.
